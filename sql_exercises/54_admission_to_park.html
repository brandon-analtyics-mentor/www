Admission to an Amusement Park Ride

Problem Description

You need to determine the last visitor that can board a roller coaster ride without exceeding the maximum combined height limit of 5000 centimeters.

Table: RideLine

+--------------+------+
| Column Name  | Type |
+--------------+------+
| visitor_id   | int  |
| visitor_name | varchar |
| height       | int  |
| position     | int  |
+--------------+------+

visitor_id column contains unique values.
This table has information about all visitors waiting to get on a roller coaster ride.
The visitor_id and position columns will contain all numbers from 1 to n, where n is the number of rows in the table.
position determines the order in which visitors will board the ride, where position=1 denotes the first visitor to board and position=n denotes the last visitor to board.
height is the height of the visitor in centimeters.

There is a line of people waiting to board a roller coaster. However, the roller coaster has a maximum combined height limit of 5000 centimeters for safety reasons, so there may be some visitors who cannot board.

Write a solution to find the visitor_name of the last visitor that can get on the ride without exceeding the height limit. The test cases are generated such that the first visitor does not exceed the height limit.

Note that only one visitor can board the ride at any given position.

Example 1

Input:
RideLine table:

+-----------+--------------+--------+----------+
| visitor_id| visitor_name | height | position |
+-----------+--------------+--------+----------+
| 1         | Alice        | 165    | 5        |
| 2         | Bob          | 180    | 3        |
| 3         | Charlie      | 175    | 4        |
| 4         | Diana        | 168    | 7        |
| 5         | Evan         | 172    | 1        |
| 6         | Frank        | 178    | 2        |
| 7         | Grace        | 155    | 6        |
+-----------+--------------+--------+----------+

Output:

+--------------+
| visitor_name |
+--------------+
| Grace        |
+--------------+

Explanation:

  • The visitors board the ride in this order: Evan, Frank, Bob, Charlie, Alice, Grace, Diana
  • The cumulative heights are: 172, 350, 530, 705, 870, 1025, 1193
  • As the height limit is 5000 centimeters, all visitors can board the ride.
  • Grace is the last visitor to board without exceeding the height limit.

Example 2

Input:
RideLine table:

+-----------+--------------+--------+----------+
| visitor_id| visitor_name | height | position |
+-----------+--------------+--------+----------+
| 1         | Alice        | 850    | 2        |
| 2         | Bob          | 910    | 4        |
| 3         | Charlie      | 1200   | 1        |
| 4         | Diana        | 760    | 5        |
| 5         | Evan         | 1030   | 3        |
| 6         | Frank        | 900    | 6        |
+-----------+--------------+--------+----------+

Output:

+--------------+
| visitor_name |
+--------------+
| Diana        |
+--------------+

Explanation:

  • The visitors board the ride in this order: Charlie, Alice, Evan, Bob, Diana, Frank
  • The cumulative heights are: 1200, 2050, 3080, 3990, 4750, 5650
  • As the height limit is 5000 centimeters, Frank cannot board the ride.
  • Diana is the last visitor to board without exceeding the height limit.
